Arbetsprocess

Utveckling och Publicering

Automatisering:

Jag har använt Node.js-verktyget Parcel för att automatisera filbearbetning och optimera filer.

  • Jag började med att kontrollera om jag hade NodeJS installerat, vilket jag hade, men jag uppdaterade till en senare version.
  • Sedan initierade jag ett nytt npm projekt genom att i terminalen ange npm init.
  • Installerade Parcel, npm install parcel --save-dev, som hjälper mig som utvecklare med automatiseringen. Genom att konfigurera Parcel har jag kunnat minifiera CSS, JavaScript och HTML samt optimera bilder automatiskt. Detta har förenklat och effektiviserat arbetsflödet och gjort det lättare att hålla koden välstrukturerad.
  • Innan publicering av sidan så körde jag npm run build för att optimera filerna.

Versionhantering:

För att hantera versioner av projektet har jag använt Git och arbetat med branches som development och main.

  • Först och främst la jag till en .gitignore fil i projektet där jag angav de foldrar och filer som jag inte vill ska visas i Git.
  • Jag initierade mitt projekt till Git och skapade två brancher, development och main.
  • Jag arbetade främst i development och sedan när jag kände mig nöjd så gjorde jag en checkout till main.

    • Demowebbplats:

    Jag har skapat den här responsiva webbplatsen som består av tre sidor:

    1. Hem: Här beskriver jag själva uppgiften.
    2. Bildgalleri: En sida med fem bilder gjorda med css grid. Jag har användt WebP, ett optimerat bildformat, och Parcel hanterar ytterligare optimering av bilderna.
    3. Arbetsprocess: Det är den här sidan där jag förklarar arbetsflödet och hur jag har konfigurerat automatiseringen.

    Jag har använt HTML, CSS och lite JavaScript. Samtliga filer är strukturerade under respektive mappar.

    Publicering:

    Jag har publicerat webbplatsen via Git och Netlify, vilket säkerställer att den alltid är uppdaterad utan manuellt arbete.

    Den här arbetsprocessen har gjort utvecklingen mer effektiv och säkerställt att allt, från automatisering till publicering, är välorganiserat och enkelt att använda för framtida projekt.
